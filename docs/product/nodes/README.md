# Nodes overview

In the **Catalog**, nodes are grouped into **Sources**, **MIDI**, **Effects**, **Routing**, and **Math**. Each guide explains what the node does and how you typically use it in the UI.

| Category | Guide |
|----------|--------|
| Sources | [sources.md](./sources.md) |
| MIDI | [midi.md](./midi.md) |
| Effects | [effects.md](./effects.md) |
| Routing | [routing.md](./routing.md) |
| Math | [math.md](./math.md) |

**Rules that apply to many nodes:**

- Outputs are wired from **source** handles; modulation and audio **inputs** are **target** handles (connect another node’s output into them).  
- When a control has a **cable** plugged in, the card often shows the **live value** and hides manual editing for that input.  
- **Singleton** nodes: only **one** Transport, one MIDI **Sync**, and one **Output** per graph—the editor prevents duplicates.
