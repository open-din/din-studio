# Review and publish

When you are ready to **commit** or **publish** work, DIN Studio uses a dedicated **Review** path instead of hiding git actions in the footer.

## Opening review

Switch the activity rail to **Review** to open the **Source Control** surface. You should see:

- A summary of **changed files**  
- Where you are in the flow (**idle**, preparing, generating, ready to commit, in progress, success)  
- Actions such as **generate** or refresh when needed  
- A **commit message** field and a clear **commit** control  

## How it fits with the rest of the UI

- The **footer** still shows quiet git branch hints; it is **not** the main commit workspace.  
- The **inspector** stays focused on **graph and node** editing, not replacing Source Control.  
- You can still use the **command palette** and other saves; Review owns the **framing** once you enter publish work.

## After a successful commit

Expect a **clear success state** and a suggested **next step** (for example continue editing or hand off), so the flow does not stop abruptly.
